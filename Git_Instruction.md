# Работа с Git

В терминале выполнить команду `git version`

Если Git установлен, появится сообщение с информацией о версии программы. Иначе будет сообщение об ошибке.

## 2. Установка Git
Загружаем последнюю версию Git с сайта https://git-scm/downloads.
Устанавливаем с настройками по умолчанию.

 ## 3. Настройка Git

 При первом использовании Git необходимо представиться. Для этого нужно ввести в терминале две команды:
 ```
 git config --global user.email "you@example.com"
 git config --global user.name "Your Name"
 ```
 
## 4. Инициализация репозитория

Для инициализации репозитория заходим в каталог с файлами (проектом) и вводим команду в терминале:

```
git init
```

## 5. Запись изменений в репозиторий

Для сохранения внесенных изменений неободимо ввести команду

```
git add "Имя директории"
```
с указанием названия соответсвующей папки. Далее вводим команду

```
git commit -m "Комментарии"
```
В кавычках указываем комментарии к данному коммиту.

В случае, если необходимо просмотреть, какие изменения были внесены в документ с момента сохранения последнего коммита, можно воспользоваться командой

```
git diff
```
После ее ввода вся удаленная информация отобразится красным цветом, а вся добавленная - зеленым.

Посмотреть статус репозитория и изменений в нем можно при помощи команды

```
git status
```

## 6. Просмотр истории коммитов

Чтобы просмотреть историю всех коммитов, нужно ввести команду

```
git log
```
Если необходимо просмотреть историю всех изменений без подробностей каждого коммита, можно запустить git с флагом **--oneline**


## 7. Перемещение между сохранениями

Для перемещения между сохранениями нужно ввести команду

```
git checkout
```
и четыре первые цифры коммита, в который нужно переместиться.

Если необходимо вернуться к последнему коммиту, можно воспользоваться командами

```
git checkout master
или
git switch -
```

## 8. Игнорирование файлов
Для того, чтобы исключить из отслеживания в репозитории определенные файлы или папки, необходимо создать там файл ***.gitignore*** и записать в него их названия или шаблоны, соответствующие таким файлам или папкам.

## 9. Создание веток в Git

Текущая ветка будет отмечена звёздочкой: **\*master**
Создать ветку можно командой:
```
git branch <имя новой ветки>
```
В результате создается новый указатель на текущий коммит.

Ветка в Git - это простой перемещаемый указатель на один из коммитов, обычно последний в цепочке коммитов.
По умолчанию имя основной ветки в Git - *master*.
Список веток в репозитории можно посмотреть с помощью команды `git branch`.

Команда `git branch -b` создает новую ветку и сразу на нее переключает.

## 10. Слияние веток и разрешение конфликтов
Слияние веток - это перенос изменений с одной ветки на другую.
При этом слияние не затрагивает сливаемую ветку, то есть она остается в том же состоянии, что позволяет нам потом продолжить работу с ней. 
Для слияния выбранной ветки с текущей нужно выполнить команду
```
git merge <название сливаемой ветки>
``` 
Очень часто во время слияния веток оказывается, что вносимые изменения удаляют или переписывают информацию в уже существующих файлах. Такая ситуация называется ***файловым конфликтом***. Git останавливает выполнение слияния, пока вы не разрешите конфликт.

Существует два способа разрешать конфликты, вот они:

1. Разрешить конфликт вручную - самостоятельно изменить конфликтные файлы, сделав их такими, какими хотим их видеть.

2. Просто выбрать один из двух файлов.

После разрешения конфликта нужно обязательно совершить ***commit слияния***!

Если возникновение конфликта стало неожиданным, можно выполнить `git merge --abort`. Эта команда прервет слияние и вернет все, как было. 

Если во время слияния не возникло конфликтов, то из вливаемой ветки извлекаются коммиты, а затем они применяются к последнему коммиту цедевой ветки. Такое слияние называется ***fast-forward***.

## 11. Удаление веток

При удалении ветки нужно быть очень осторожным, поскольку все изменения, которые не были добавлены в коммит, будут утеряны.

С операцией удаления над ветками справляется уже привычная команда git branch с параметром -d:
```
git branch -d <имя ветки>
```
Для корректного удаления нужно помнить несколько правил, чтобы не получить ошибки:

* Нельзя удалить ветку, в которой вы находитесь. Git выкинет ошибку и не произведет удаление. Следовательно, нужно перейти на другую ветку.

* Git не позволит удалить ветку, у которой есть несохраненные изменения. Так мы избегаем ситуации, когда часть написанной информации будет безвозвратно утеряна. Если же мы уверены, что изменения в этой версии не нужны и их можно смело удалять, то вместо флага -d используем -D:
```
git branch -D
```

## 12 Создание удаленного репозитория
Создадим новый репозиторий на GitHub путём заполения формы указав:
    - название репозитория;
    - тип: публичный или закрыттый;
    - создание файла с описанием READ.ME;

## 13 Добавление удаленного репозитория. Команда git remote add origin
Чтобы добавить удаленный репозиторий и присвоить ему название, которое используется для удобства как сокращение адреса, выполняем команду
```    
    git remote add origin <URL>
```    
По умолчанию удаленный репозиторий называется origin, но можно присваивать любое название.
Ссылочные объекты у подключаемого удаленного репозитория будут скопированы.    

## 14 Просмотр удаленного репозитория
Для того чтобы просмотреть список названий удаленных репозиториев, которые были добавлены, используем команду:
```
git remote
```
Указав ключ -v, можно просмотреть, какие адреса в добавленных удаленных репозиториях используются для чтения и записи.

```
git remote -v              # показать список удалённых репозиториев, связанных с локальным
git remote remove origin   # убрать привязку удалённого репозитория с сокр. именем origin
git remote add origin URL  # добавить удалённый репозиторий (с сокр. именем origin) с указанным URL
git remote rm origin       # удалить привязку удалённого репозитория
git remote show origin     # получить данные об удалённом репозитории с сокращенным именем origin
```
